«Rileggi totali» applica i pattern aggiornati (es. Totale parcella / proforma) alle righe documento che hanno ancora il testo estratto; le righe senza testo vanno reimportate dalla sezione Proforma/parcelle.

L’Excel da «Estratto conto» / «Pagamenti» importa solo bonifici in avere, non movimenti in dare. Le proforma/parcelle in dare arrivano dagli upload PDF. Due righe documento simili di solito sono due PDF con nome file diverso ma stesso numero estratto: il secondo upload viene ignorato (messaggio «Già presente in libro …»). Il libro considera univoca la coppia data documento + numero (stesso numero in anni diversi è ammesso se le date differiscono). Badge Doppio se la stessa terna data+numero+tipo compare più volte. Righe da togliere: Admin → ACCOUNTS → Movimenti partitario consulente–azienda; dopo eliminazioni i saldi si ricalcolano da soli. Bonifici in avere creati per errore dal vecchio import Excel «riepilogo PROFORMA» (riferimento tipo PARCELLA 182|data|importo accanto alle parcelle già in dare): da terminale, con l’ID azienda corretto, prima anteprima poi esecuzione — python manage.py rimuovi_bonifici_import_excel_studio --azienda-id ID --solo-parcella-proforma-sintetici poi python manage.py rimuovi_bonifici_import_excel_studio --azienda-id ID --solo-parcella-proforma-sintetici --execute (non tocca i bonifici bancari reali né l’import «estratto conto»). Per rimuovere tutti i bonifici tracciati Excel di quell’azienda usare il comando senza --solo-parcella-proforma-sintetici (vedi deploy/PROCEDURA_DEPLOY.md).
