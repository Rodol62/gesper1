«Rileggi totali» applica i pattern aggiornati (es. Totale parcella / proforma) alle righe documento che hanno ancora il testo estratto; le righe senza testo vanno reimportate dalla sezione Proforma/parcelle.

I bonifici in avere si registrano in Pagamenti (manuale o distinte PDF); le proforma/parcelle in dare dagli upload PDF in Proforma. Due righe documento simili di solito sono due PDF con nome file diverso ma stesso numero estratto: il secondo upload viene ignorato (messaggio «Già presente in libro …»). Il libro considera univoca la coppia data documento + numero (stesso numero in anni diversi è ammesso se le date differiscono). Badge Doppio se la stessa terna data+numero+tipo compare più volte. Righe da togliere: utenti admin hanno in tabella la colonna Admin con link Modifica/Elimina (Django Admin); altrimenti Admin → ACCOUNTS → Movimenti partitario consulente–azienda. Dopo eliminazioni i saldi si ricalcolano da soli. Pulizia massiva di bonifici sintetici errati (riferimento tipo PARCELLA …|data|importo duplicati rispetto al dare): da terminale, con l’ID azienda corretto, prima anteprima poi esecuzione — python manage.py rimuovi_bonifici_import_excel_studio --azienda-id ID --solo-parcella-proforma-sintetici poi python manage.py rimuovi_bonifici_import_excel_studio --azienda-id ID --solo-parcella-proforma-sintetici --execute (non tocca i bonifici bancari reali registrati manualmente). Per un intervento più ampio sull’azienda vedi lo stesso comando senza --solo-parcella-proforma-sintetici in deploy/PROCEDURA_DEPLOY.md.
